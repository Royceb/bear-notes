Archiver Bot

The purpose of this bot is to save time and money by archiving channels that meet a certain criteria.


  • Archive conversations with 0 members
  • Archive channels with no activity in 6 months and with 1 member
  • For channels with more than 1 member and no activity in 6 months - send message to conversation that it will be archived. If someone doesn’t object within X amount of time or if there is no objection, bot will remove all members then archive channel (by removing all members people will not be aggressively notified about the channels being archived)

Bolt

Bolt is a Javascript library that implements and provides easy to use interfaces to most of Slack API’s.


Slack App Setup

Clone https://github.com/RobotsAndPencils/sample-slack and follow the instructions in the README.md.


You should have a Slack app created at Slack API: Applications | Slack. Make sure to include a bot with this app. The bot will be used to find channels to archive and archive them.


Install the app into a workspace.


Archive conversations with 0 members

There are multiple ways to start the archival process:

  1. Via App Home
  2. Via a Slash Command

We will use App home for this case. We could use the Home tab or the Messages tab.


Using the Home tab, we can display buttons to start the archival process.

Using the Messages tab, we can display buttons/blocks directly in the chat.


Slack triggers an event, app_home_opened when the App Home is opened. We could use this event to send a message as well, like described directly above.


The bot will not have to be installed or added to public conversations manually - it can join itself into a public conversation. However, bots cannot add themselves to private conversations using a bot token.

Make sure to only include public and private channels . We do not want to include 1:1/DM conversations or group DMs.


Once the bot gets a list of conversations in a workspace, it can figure out which ones are archivable. For now, conversations are archivable only if they do not have any members.


The bot will aggregate the conversations that match this criteria, and then archive the channel using: 


bolt.client.conversations.archive({
	token,
	channel
})



Archive channels with no activity in 6 months and with 1 member

This is very similar to the above, we just change our criteria for archival by including conversations with 1 user and no activity in 6 months.


We can use the conversations history of messages to determine if there is stale activity or not. 


bolt.client.conversations.history({
    token,
    channel,
    oldest: sixMonthsAgo(),
});


By using the oldest argument, we can load conversation history that started 6 months ago.


We must make sure to filter/exclude any message that was not sent from a real human user.


If the length of the history is greater than 0, that means there has been activity in the last 6 months. If less than 0, there has been no activity and the channel can be archived.
